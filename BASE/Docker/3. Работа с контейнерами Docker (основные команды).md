1. Команда отображения списка активных контейнеров в системе
`docker ps`позволяет получить различную информацию о работающих контейнерах, включая их ID, имена, статус, порты, использование ресурсов.

Отображение списка всех контейнеров(включая остановленные):
```
docker ps -a 
```
Отображение размера занимаемого дискового пространства:
```
docker ps -s 
```

***

Запуск контейнера на основе образа.
```
docker run hello-world
```

Запуск контейнера в фоновом режиме, проброс порта с хоста на контейнер и передача переменной окружения:
```
docker run -d -p 8080:80 -e ENV_VAR=value nginx
```

Запуск контейнера с ограничением ресурсов, в данном примере — количества оперативной памяти:
```
docker run --memory=2g myimage
```

***

Запускает команду внутри запущенного контейнера.   
`docker exec container_id command`
Пример: посмотреть список файлов внутри контейнера: 
```
docker exec 123456abcdef ls
```

Пример:  запустить шелл-оболочку системы (например, bash) в контейнере,  -it включит терминал для возможности ввода команд  в контейнере: 
```
docker exec -it 123456abcdef bash
```

***

Отобразить логи контейнера.  
```
docker logs container_id
```
Копировать файлы или папки между локальной машиной и контейнером.  
```
docker cp container_id:/path/to/file local_path
```
Отобразить список сетей Docker.
```
docker network ls
```
Создать новую сеть Docker.
```
docker network create
```

***

Остановка одного работающего контейнера:
```
docker stop mycontainer
```

Запуск одного остановленного контейнера:
```
docker start mycontainer
```

Перезапуск одного контейнера:
```
docker restart mycontainer
```

***

Удаление одного контейнера:
```
docker rm mycontainer 
```

Удаление нескольких:
```
docker rm first_container second_container third_container
```

Принудительное удаление (позволяет удалить контейнер, который выполняется в данный момент):
```
docker rm -f mycontainer
```
